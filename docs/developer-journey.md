# Customer Journey Map: Застройщик, предлагающий меблированные квартиры

## Профиль персонажа

### **ООО "Новый Дом"**
- **Тип компании**: Средний застройщик жилья комфорт-класса, реализующий 2-3 жилых комплекса одновременно
- **Масштаб**: Ежегодно сдает около 600-800 квартир, из которых 20-25% предлагаются с готовой меблировкой
- **Стратегия развития**: Увеличение доли меблированных квартир до 40% за 2 года для повышения маржинальности проектов
- **Ключевой представитель**: Андрей, 41 год, руководитель отдела комплектации и отделки, отвечает за выбор партнеров-поставщиков мебели

### Ключевые характеристики
- **Работа с большими объемами**: Необходимость одновременно комплектовать 100+ квартир различных планировок
- **Стандартизация решений**: Ограниченное количество вариантов планировок (10-15 типов) с типовыми решениями для каждой
- **Ограничения по времени**: Жесткие сроки сдачи объектов, требующие быстрого проектирования и производства мебели
- **Ценовое давление**: Необходимость удерживать стоимость меблировки в рамках 7-10% от стоимости квартиры

## Этапы Customer Journey

### 1. **Осведомленность**

#### Действия
- Обнаруживает информацию об AI-решении для проектирования мебели в отраслевом журнале по недвижимости
- Замечает упоминание в презентации на строительной выставке в секции "умные решения для девелоперов"
- Получает рекомендацию от партнера-производителя мебели
- Изучает информацию на сайте, обращая внимание на раздел B2B-решений

#### Мысли
- "Может ли это решение масштабироваться на сотни квартир одновременно?"
- "Насколько легко интегрировать с нашими планировками из AutoCAD/Revit?"
- "Сколько это сэкономит во времени и бюджете при комплектации наших объектов?"
- "Сможем ли мы создать каталог типовых решений для наших планировок?"

#### Эмоции
- Заинтересованность в оптимизации процессов меблировки
- Скептицизм относительно реальной применимости для массового жилья
- Надежда на сокращение сроков и затрат на проектирование
- Осторожность по отношению к новым технологическим решениям

#### Болевые точки
- Сложность координации между архитекторами, дизайнерами и производителями мебели
- Текущие решения требуют слишком много ручной работы дизайнеров
- Высокие затраты на создание технической документации для производства
- Длительный цикл от проектирования до производства замедляет сдачу объектов

#### Возможности
- Предоставить демонстрацию работы на реальных планировках застройщика
- Показать кейсы успешного внедрения в девелоперских проектах
- Представить калькулятор ROI с учетом масштаба проектов
- Предложить тестирование на одном из объектов без обязательств

- Организует демонстрационную встречу с командой проектировщиков и закупщиков

- Консультируется с текущими поставщиками мебели о возможности интеграции с их производством
- Проводит расчет потенциальной экономии времени и бюджета

#### Мысли
- "Какие технические требования необходимы для внедрения?"
- "Потребуется ли обучение наших сотрудников или отдельная команда?"
- "Как это решение впишется в наш текущий процесс проектирования и сдачи объектов?"
- "Какие риски могут возникнуть при переходе на новую систему?"

#### Эмоции
- Воодушевление от возможности оптимизировать процессы
- Беспокойство о сложности внедрения
- Осторожный оптимизм относительно потенциальной экономии
- Интерес к инновационным подходам как конкурентному преимуществу

#### Болевые точки
- Необходимость совместимости с существующими BIM/CAD системами
- Опасения по поводу надежности нового решения при массовом использовании
- Вопросы интеграции с текущими процессами проектирования
- Неуверенность в готовности производителей мебели работать с новым форматом данных

#### Возможности
- Разработать процесс интеграции с системами проектирования застройщика
- Предложить пилотный проект с ограниченным количеством типовых планировок
- Предоставить документацию по API для интеграции с системами застройщика
- Организовать семинар для проектировщиков и поставщиков мебели

### 3. **Тестирование**

#### Действия
- Загружает 3-5 типовых планировок для тестового проектирования
- Формирует рабочую группу из представителей разных отделов для оценки системы
- Проводит сравнительный анализ времени проектирования с текущим подходом
- Запрашивает тестовые образцы технической документации для передачи производителям

#### Мысли
- "Насколько точны автоматически генерируемые спецификации?"
- "Можем ли мы создать библиотеку стандартных решений для каждого типа квартир?"
- "Как быстро система позволяет вносить массовые изменения при корректировке планировок?"
- "Насколько качественны визуализации для маркетинговых материалов?"

#### Эмоции
- Удивление от скорости генерации проектов
- Удовлетворение от точности технических спецификаций
- Воодушевление от потенциала визуализаций для отдела продаж
- Некоторое напряжение при выявлении ограничений системы

#### Болевые точки
- Необходимость доработки некоторых аспектов системы под специфику массового жилья
- Сложности с одновременной работой над множеством однотипных планировок
- Проблемы с интеграцией некоторых нестандартных элементов
- Ограничения по кастомизации под брендбук застройщика

#### Возможности
- Разработать функциональность массового проектирования для однотипных квартир
- Создать специальный интерфейс для работы с многоквартирными проектами
- Внедрить систему управления вариантами отделки и меблировки
- Предложить интеграцию с системами визуализации для маркетинговых материалов

## Метрики успеха

### Количественные показатели

| Метрика | До внедрения | После внедрения | Изменение |
|---------|--------------|-----------------|-----------|
| Время проектирования типовой квартиры | 6-8 часов | 1-2 часа | -75% |
| Цикл от проектирования до производства | 4-6 недель | 2-3 недели | -50% |
| Количество корректировок на этапе производства | 30-35% проектов | 5-10% проектов | -75% |
| Затраты на проектирование мебели на м² | $4.5-5.5 | $1.8-2.2 | -60% |
| Скорость визуализации для маркетинга | 3-4 дня | 4-6 часов | -85% |
| Объем меблированных квартир в портфеле | 20-25% | 35-40% | +75% |

### Качественные показатели

- Повышение точности проектов и соответствия техническим требованиям
- Улучшение визуализаций для маркетинговых материалов
- Возможность быстрой адаптации к изменениям планировок
- Повышение привлекательности предложения для покупателей квартир
- Укрепление партнерских отношений с производителями мебели
- Создание нового конкурентного преимущества на рынке недвижимости
